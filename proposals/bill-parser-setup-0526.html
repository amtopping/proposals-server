STEP 01
Auto-Managed Properties
These properties are created and managed by the system. You do not need to do anything on this page.
✋
No action needed on this page.
All properties listed below are created automatically the first time the system runs. Do not add them manually — Google Apps Script will not save a property with an empty value.
QUEUE_STATUSAuto

Whether the standard bill processing queue is currently running or idle.

QUEUE_FILESAuto

JSON array of files currently in the processing queue.

QUEUE_TOTALAuto

Total file count for the current run.

QUEUE_PROCESSEDAuto

How many files have been processed in the current run.

QUEUE_FAILEDAuto

How many files failed in the current run.

QUEUE_MODEAuto

Whether the queue is running in normal or retry mode.

QUEUE_STARTED_ATAuto

Timestamp for when the current run started.

ME_QUEUE_STATUSAuto

Status flag for the multi-entity file queue.

ME_QUEUE_FILESAuto

JSON array of multi-entity files pending processing.

ME_QUEUE_TOTALAuto

Total multi-entity file count for the current run.

ME_QUEUE_PROCESSEDAuto

How many multi-entity files have been processed.

ME_QUEUE_FAILEDAuto

How many multi-entity files failed.

JSON array of Drive file IDs already processed. Prevents duplicates.

TEMPLATES_JSONAuto

Template thumbnail registry. Populated when you run Render Template Thumbnails from the Bill Parser menu.

STEP 02
Anthropic API Key & Alert Emails
One API key powers both extraction passes. Set your primary alert email and optionally a secondary one.
CLAUDE_API_KEYRequired

The Anthropic API key that powers both extraction passes — primary parsing and secondary validation. This is the only AI key the system needs.

Where to get it:
1. Go to console.anthropic.com
2. Sign in or create a free account
3. Click API Keys in the left menu
4. Click Create Key, give it a name like "Bill Parser"
5. Copy the key (starts with sk-ant-api03-...) — you only see it once
6. Paste it as the value for CLAUDE_API_KEY
ALERT_EMAIL_PRIMARYRequired

The main email address that receives all daily summaries and error alerts.

Set this to: Your main email address. Every alert and daily digest goes here.
ALERT_EMAIL_SECONDARYOptional

A second email that also receives all alerts and daily summaries. Leave this out entirely if you only need one recipient.

Set this to: One additional email address. Example: manager@yourcompany.com

If you skip this property, only ALERT_EMAIL_PRIMARY receives alerts. You can add it later at any time.
STEP 03
Google IDs
The Sheet ID and Drive Folder ID tell the system where to read bills from and where to write results.
SHEET_IDRequired

The Google Sheet where parsed bill data will be written and the dashboard reads from.

How to find it:
1. Open your Google Sheet
2. Look at the URL — it will look like:
docs.google.com/spreadsheets/d/1kG5aQf_hd-.../edit
3. The long string between /d/ and /edit is your Sheet ID
4. Paste it as the value for SHEET_ID

Note: This may also auto-populate when you run Setup Sheets for the first time.
DRIVE_FOLDER_IDRequired

The Google Drive folder where utility bill PDFs will be dropped for processing.

How to set it up:
1. Go to drive.google.com
2. Create a new folder (e.g. "GPM Utility Bills")
3. Open that folder and look at the URL:
drive.google.com/drive/folders/1s3PIbHoz2IP...
4. The long string at the end is your Folder ID
5. Paste it as the value for DRIVE_FOLDER_ID
STEP 04
Carry Over Values
These two values will be provided by Keyona. Copy them in exactly as given.
FINGERPRINTS_JSONCarry Over

Known bill layout signatures for each utility provider. Enables fast, accurate parsing of bills you have seen before.

Paste the value exactly as provided by Keyona. Do not modify it.
PROVIDERS_JSONCarry Over

All known utility providers (electricity, gas, water) and their account structures. Allows the system to categorize bills by type automatically.

Paste the value exactly as provided by Keyona. Do not modify it.
