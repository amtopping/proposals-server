Step01

What You're Setting Up

Before touching anything, here's the full picture of what this system is and what setup looks like.

What it does
Automated Bill Parsing
You drop utility bill PDFs into a Google Drive folder. The system reads them, extracts the key data (usage, cost, dates), and writes each one to the right tab in your Google Sheet automatically.
How it runs
Runs Every 15 Minutes
A time-driven trigger checks your Drive folder every 15 minutes. Any new files get picked up and processed without you doing anything. You receive a daily email summary of what was processed.
Where results go
Three Output Tabs
Parsed bills are written to one of three tabs: Electricity, NaturalGas, or Water. The dashboard gives you a live view of processing status.
When something needs review
Review Queue
If the system can't confidently parse a bill it goes to the Review Queue tab for manual review. You'll get an email alert when this happens.

System Lifecycle
8 steps · click to expand
01
PDF Drop
Intake
›
Bryan or Adrian uploads utility bill PDFs to the designated Google Drive folder. No naming convention required — the system reads file content, not filenames. Files can be dropped at any time and will be picked up on the next trigger cycle.
02
Trigger Fires
Apps Script
›
Every 15 minutes, a time-driven Google Apps Script trigger scans the Drive folder for new PDFs. It compares incoming files against a processed file cache and ignores duplicates. New files are classified and loaded into the queue.
03
Queue & Classify
Apps Script
›
Standard bills go into the main queue, multi-entity bills go into a separate queue. The Queue Manager batches files one at a time to stay within Google's execution limits, using continuation triggers to chain runs automatically for large batches.
04
Pass 1 — Extract
AI Engine
›
The PDF is sent to the Claude API for primary extraction. Claude reads the bill and extracts bill type, account number, property address, billing period dates, usage amount, total cost, demand charges, and whether the bill is an estimate. Large files over 800KB get an automatic delay between passes.
05
Pass 2 — Validate
AI Engine
›
A second independent Claude API call re-reads the same bill without seeing Pass 1's output. Two passes catch OCR ambiguities, formatting edge cases, and occasional misreads — especially on usage figures and billing period dates where a one-digit error would compound over time.
06
Conflict Check
Verifier
›
The Verifier compares every field between Pass 1 and Pass 2. If both agree, data goes to the Google Sheet. If any field conflicts, the bill goes to the Review Queue and Bryan receives an email showing exactly which fields disagreed. Nothing ambiguous goes into the sheet automatically.
07
Sheet Write
Output
›
Verified data is written to the correct tab — Electricity, NaturalGas, or Water — based on the bill type extracted by the AI. Each row includes the full extracted fields, a Duplicate flag, a verification formula, and a link back to the original PDF in Drive.
08
Daily Summary
Delivery
›
At 6pm every day, Bryan and Adrian receive an email digest showing how many bills were processed, how many are pending in the Review Queue, and the current collection status for each account. The dashboard is available anytime at the web app URL. No manual data entry required.
Setup takes about 15 minutes. You'll move through four configuration steps: entering your own script properties, pasting in two carry-over values from Keyona, reviewing the auto-managed properties (no action needed), and then running the setup steps from the Bill Parser menu in your Google Sheet.
Step02

Your Script Properties

These are the values only you can provide. Open Script Properties in Apps Script and add each one below.

How to open Script Properties: In your Google Sheet, click Extensions → Apps Script. In the editor, click the gear icon (Project Settings) in the left sidebar. Scroll down to Script Properties and click Edit script properties.
Script Properties — What a correctly configured project looks like
CLAUDE_API_KEYRequired

The Anthropic API key that powers both bill extraction passes. This is the only AI key the system needs.

Where to get it:
1. Go to console.anthropic.com
2. Sign in or create a free account
3. Click API Keys in the left menu
4. Click Create Key, name it "Bill Parser"
5. Copy the key (starts with sk-ant-api03-...) — shown once only
6. Paste as the value for CLAUDE_API_KEY
ALERT_EMAIL_PRIMARYRequired

The main email that receives all daily summaries and error alerts.

Set this to: Your main email address. Every alert and daily digest goes here.
ALERT_EMAIL_SECONDARYOptional

A second email that also receives all alerts and summaries. Skip entirely if you only need one recipient — do not add this property with an empty value.

Set this to: One additional email. Example: manager@yourcompany.com

You can always add this later if needed.
SHEET_IDRequired

The Google Sheet where parsed bill data is written and the dashboard reads from.

How to find it:
1. Open your Google Sheet
2. Look at the URL:
docs.google.com/spreadsheets/d/[THIS PART]/edit
3. Copy the long string between /d/ and /edit
4. Paste as the value for SHEET_ID
DRIVE_FOLDER_IDRequired

The Google Drive folder where utility bill PDFs are dropped for processing.

How to find it:
1. Open the Drive folder shared with you, or create a new one
2. Look at the URL:
drive.google.com/drive/folders/[THIS PART]
3. Copy the long string at the end
4. Paste as the value for DRIVE_FOLDER_ID
Step03

Carry Over Values

These two values come directly from Keyona. Do not look these up yourself — paste them exactly as provided.

FINGERPRINTS_JSONCarry Over

Known bill layout signatures for each utility provider. Enables fast, accurate parsing of bills the system has seen before.

Paste the value exactly as provided by Keyona. Do not modify it.
PROVIDERS_JSONCarry Over

All known utility providers and their account structures. Allows the system to categorize bills by type automatically.

Paste the value exactly as provided by Keyona. Do not modify it.

Step05

Menu Setup

Once your Script Properties are saved, go back to your Google Sheet. After refreshing, a Bill Parser menu will appear in the top menu bar. Run these three steps in order.

What your Google Sheet tabs will look like after Setup Sheets
1
Run Setup Sheets
In your Google Sheet, click Bill Parser → Setup Sheets. This creates all required tabs: Electricity, NaturalGas, Water, Review Queue, Run Status, and more. You only need to do this once. A confirmation dialog will appear when done.
2
Run Setup Trigger
Click Bill Parser → Setup Trigger. This installs two automated triggers: one that checks your Drive folder for new bills every 15 minutes, and one that sends a daily summary email at 6pm. Without this step the system will not run automatically.
3
Deploy the Web App
Go back to Apps Script. Click Deploy → New Deployment. Set type to Web App. Set Execute as to Me and Who has access to Anyone. Click Deploy and copy the /exec URL — that is your dashboard link.
Full Bill Parser Menu Reference
Run Now
Manually triggers a processing run immediately. Scans the Drive folder for new PDFs and processes them. Use this when you've just dropped new bills and don't want to wait for the 15-minute trigger.
Retry Pending Review Queue
Re-runs processing on any bills in the Review Queue with PENDING status. Use this after fixing a setup issue or when asked to retry a failed bill.
Cancel Run
Stops any active processing run and resets the queue to idle. Use this if a run appears stuck or if you need to interrupt processing for any reason.
Generate Final File
Exports a clean spreadsheet of all non-duplicate bills for a specific account and emails it to you. Use when you need a formatted file for reporting. Do not run this while a processing run is active.
Delete a Bill
Removes a specific bill row from the sheet by tab type and row number, and moves it to the Trash tab. Use to clean up test entries or incorrectly parsed rows.
Setup Sheets
Creates all required tabs in the Google Sheet. Run once during initial setup. Safe to re-run if a tab is accidentally deleted.
Setup Trigger
Installs the 15-minute processing trigger and the 6pm daily summary trigger. Run once during initial setup.
Send Daily Summary Now
Sends the daily summary email immediately rather than waiting for 6pm. Useful for testing your alert email or getting an on-demand status report.
Clear Processed Cache
Resets the list of processed file IDs. Use only if you need the system to re-evaluate files it has already seen. Keyona should confirm before you run this.
Show Dashboard URL
Displays your web app URL in a dialog. Use this if you lose your dashboard link and need to find it again quickly.
