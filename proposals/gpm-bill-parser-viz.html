System Lifecycle
8 steps · click to expand
01
PDF Drop
Intake
›
Bryan or Adrian uploads utility bill PDFs to the designated Google Drive folder. No naming convention required — the system reads file content, not filenames. Files can be dropped at any time and will be picked up on the next trigger cycle.
02
Trigger Fires
Apps Script
›
Every 15 minutes, a time-driven Google Apps Script trigger scans the Drive folder for new PDFs. It compares incoming files against a processed file cache and ignores duplicates. New files are classified as standard or multi-entity and loaded into the queue.
03
Queue & Classify
Apps Script
›
Files are classified before processing. Standard bills go into the main queue, multi-entity bills go into a separate queue. The Queue Manager batches files one at a time to stay within Google's execution limits, using continuation triggers to chain runs automatically for large batches.
04
Pass 1 — Extract
AI Engine
›
The PDF is sent to the Claude API for primary extraction. Claude reads the bill and extracts: bill type, account number, property address, billing period dates, usage amount, total cost, demand charges (for electricity), and whether the bill is an estimate. Large files over 800KB get an automatic delay between passes to stay within rate limits.
05
Pass 2 — Validate
AI Engine
›
A second independent Claude API call re-reads the same bill without seeing Pass 1's output. Two passes catch OCR ambiguities, formatting edge cases, and the occasional misread — especially on usage figures and billing period dates where a one-digit error would compound over time.
06
Conflict Check
Verifier
›
The Verifier compares every extracted field between Pass 1 and Pass 2. If both agree, data goes to the Google Sheet. If any field conflicts, the bill goes to the Review Queue and Bryan receives an email showing exactly which fields disagreed. Nothing ambiguous goes into the sheet automatically.
07
Sheet Write
Output
›
Verified data is written to the correct tab — Electricity, NaturalGas, or Water — based on the bill type extracted by the AI. Each row includes the full extracted fields, a Duplicate flag, a verification formula, and a link back to the original PDF in Drive.
08
Daily Summary
Delivery
›
At 6pm every day, Bryan and Adrian receive an email digest showing how many bills were processed, how many are pending in the Review Queue, and the current collection status for each account. The dashboard is available anytime at the web app URL. No manual data entry required.
