Step01

How This System Works

A quick orientation before you build. Understand what the system does and what it produces before touching any configuration.

The loop
Fathom in, draft out
After every Fathom-recorded meeting, the script fetches the transcript and summary via API, sends both to Claude, and creates a follow-up draft in Gmail. Runs every 5 minutes automatically.
Human in the loop
Nothing sends automatically
Every output lands in Gmail as a draft. You review, adjust if needed, and send when ready. A digest email also notifies you each time a meeting is processed with links to the recording and draft.
9 files
What gets built
Config · FathomClient · MeetingProcessor · ProcessingLog · OutputDispatcher · TriggerSetup · FathomDiagnostic · Dashboard (HTML) · appsscript.json. Each file has one job.
Fully configurable
No hardcoded values
Every client-specific value — name, email, company, tone, colors — lives in Script Properties. The code is generic. Your configuration makes it yours.
Staging mode is built in. Set a STAGING_EMAIL Script Property and all drafts and digests route there instead of real recipients.
Step02

What You Need First

Gather these before touching any code. All are free or pay-as-you-go.

Fathom API KeyRequired

Used to fetch meeting transcripts and summaries via the Fathom external API.

app.fathom.video → Settings → Integrations → API → Generate key → copy it.
Anthropic API KeyRequired

Sends the structured transcript to Claude and receives the follow-up draft. Model defaults to claude-sonnet-4-5. Typical cost under $5/month.

console.anthropic.com → API Keys → Create new secret key → copy it.
Google AccountRequired

Gmail + Google Drive + Apps Script + Calendar. All standard with any Google account.

Staging Email AddressRecommended

A secondary email where drafts and digests route during testing. Your own Gmail works.

Set this first. Easier to turn staging off than explain a test email to a real contact.
Step03

Create the Apps Script Project

Set up a blank GAS project, name it, and configure the manifest before adding any files.

Do this while logged into the correct Google account. The script creates Gmail drafts and sends digest emails from whichever account owns the project.
1
Open Google Apps Script
Go to script.google.com and click New project. Rename it to Post-Meeting Intelligence by clicking "Untitled project" at the top.
2
Show the manifest file
Project Settings (gear icon) → check Show "appsscript.json" manifest file in editor.
3
Replace the manifest
Click appsscript.json. Delete everything and paste the manifest from Step 04 below.

Step06

Deploy the Dashboard

Deploy as a web app to get your dashboard URL, then install the automatic trigger.

1
Deploy as web app
Deploy (top right) → New deployment → gear icon → Web app. Set Execute as to "Me" and Who has access to "Only myself." Click Deploy.
2
Copy the web app URL
Copy and save the URL shown after deploying. Format: https://script.google.com/macros/s/[ID]/exec. This is your dashboard.
3
Authorize the script
Open the URL. Click through the Google authorization flow. If you see "unverified app," click Advanced → Go to [project name].
4
Run the diagnostic
On the dashboard, click Run Diagnostic. Confirms your Fathom key works and Claude is reachable before the trigger fires automatically.
5
Install the trigger
In the GAS editor, select installTrigger from the function dropdown and click Run. Verify it appears under Triggers in the left sidebar with handler runEngine.
Every time you change code, redeploy. Deploy → Manage deployments → edit → save new version.
