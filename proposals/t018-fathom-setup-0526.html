Step01

How This System Works

Understand what gets built before you start. This system has 9 files, one web app dashboard, and two API connections.

The loop
Fathom in, draft out
After every Fathom-recorded meeting, the script fetches the transcript and Fathom-generated summary via API, sends both to Claude, and creates a follow-up draft in Gmail. Runs every 5 minutes automatically.
Human in the loop
Nothing sends automatically
Every output lands in Gmail as a draft. You review, adjust if needed, and send when ready. A digest email also notifies you each time a meeting is processed with links to the Fathom recording and the draft.
9 files
What gets built
Config · FathomClient · MeetingProcessor · ProcessingLog · OutputDispatcher · TriggerSetup · FathomDiagnostic · Dashboard (HTML) · appsscript.json. Each file has one job.
Dashboard
Web app control panel
A deployed GAS web app: run now, load past meetings, reprocess specific meetings, and view the processing log. No Google Sheets required after setup.
Staging mode is built in. Set a STAGING_EMAIL script property and all drafts and digests route to that address instead of your real recipients.
Step02

What You Need First

Gather these before touching any code. All are free or pay-as-you-go.

Fathom API KeyRequired

Used to fetch meeting transcripts and summaries. The system uses the Fathom external API at https://api.fathom.ai/external/v1.

app.fathom.video → Settings → Integrations → API → Generate key → copy it. Paste into Script Properties in Step 05.
Anthropic API KeyRequired

Sends the structured transcript to Claude and receives the follow-up draft. Model is claude-sonnet-4-5. Typical cost under $5/month for regular VC meeting volume.

console.anthropic.com → API Keys → Create new secret key → copy it. Paste into Script Properties in Step 05.
Google AccountRequired

Gmail + Google Drive + Apps Script + Calendar. All standard with any Google account. Calendar is used as a fallback to identify attendees when Fathom does not have invitee emails.

Staging Email AddressRecommended

A secondary email (your own Gmail works) where all drafts and digests route during testing.

Set this first. Easier to turn staging off than explain a test email to a real contact.
